이제까지 GitHub를 개발 소스의 저장소(git)로만 활용해 왔었다. 참고로 GitHub는 개인 무료 사용자에게도 Private 저장소를 제공한다(2019). 글쓰기 훈련으로 시작한 블로그는 TiStory(티스토리)를 시작으로 해서 최근에는 테마를 수정한 Tumblr(텀블러)를 사용했었다. 티스토리와 같은 국내 블로그를 선호하지 않는 이유는 제한 사항이 너무 많아서였다. 이를테면 신고와 같은 검열기능과 블라인드 처리 방식. 또한 텀블러는 무분별하게 사용되거나 너무 모바일 환경에 치우쳐서 글쓰기 위주의 블로그 기능이 편하지 않아 모바일 화면과 PC의 웹 화면이 다르게 보이는 경우가 많았다. 

GitHub Pages(Jekyll) 또한 [Markdown(마크다운)](https://gist.github.com/ihoneymon/652be052a0727ad59601) 문법을 이용한 페이지, 글쓰기가 초보에게는 어려운 단점이 있지만 GitHub를 사용하는 사람은 대부분 개발자이고 WebSite만을 위해 GitHub를 가입하는 경우는 드물기 때문에 사이트 관리가 그렇게 어렵지는 않을 것이다. 익숙해지면 마크다운처럼 편한 것도 없다. 텀블러 때도 마크다운으로 포스팅 했었다. 매일 GitHub와 동거하는 나로서는 GitHub Pages에서 블로그(WebSite)를 운영하는 것이 좀 더 효율적이다.
